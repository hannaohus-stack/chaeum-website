Foundation Series — F-02

Claude 설치 및 기본 세팅

읽는 시간 약 12분 · 무료 공개 · 다음 세션 → F-03 MCP 연결 방법

Claude를 브랜드 운영 도구로 제대로 쓰려면 단순히 채팅창에서 쓰는 것과는 다릅니다. Projects 기능을 활성화하고, 브랜드 정보를 학습시키는 환경을 만드는 것이 시작점입니다. 이 세션에서는 가입부터 브랜드 운영에 최적화된 세팅까지 단계별로 알려드립니다.

2.1
플랜 선택 — 무엇을 써야 하는가

claude.ai에는 세 가지 플랜이 있습니다. 브랜드 운영 목적이라면 플랜 선택이 중요합니다.

Free
$0
기본 대화 가능. Projects 기능 제한적. 브랜드 운영 용도로는 부족합니다.
Pro
$20 / mo
Projects 완전 사용 가능. 긴 문서 처리, 우선 접근권. 1인 브랜드 운영에 충분합니다.
채움 추천
Team
$30 / mo
팀원 공유, 관리 기능 추가. 2인 이상 브랜드 운영 시 고려하세요.
Pro 플랜이 필요한 이유
Projects에 브랜드 파일을 올려두고 지속적으로 학습시키려면 Pro가 필수입니다. Free 플랜은 Projects 사용량이 제한되어 실무 활용이 어렵습니다. 월 2만 5천원 정도로 브랜드 운영 어시스턴트를 갖는 투자입니다.
2.2
가입 및 초기 설정
01
claude.ai 접속 후 회원가입
Google 계정 또는 이메일로 가입합니다. 이메일 인증 후 바로 사용 가능합니다.
팁 — 브랜드 운영용 Google 계정을 별도로 만들어 연결하면 관리가 편합니다.
02
Pro 플랜으로 업그레이드
우측 상단 프로필 → Settings → Plans & Usage → Upgrade to Pro. 카드 결제 후 즉시 적용됩니다.
03
언어 설정 확인
Settings → Language에서 Korean으로 설정하면 한국어 응답이 더 자연스럽게 나옵니다. 단, 영어로 두고 한국어로 요청해도 잘 작동합니다.
2.3
Projects 생성 — 브랜드의 두뇌 만들기

Projects는 Claude가 브랜드 정보를 지속적으로 기억하는 공간입니다. 일반 대화와 달리 창을 닫아도 학습 내용이 유지됩니다.

01
좌측 사이드바 → Projects → New Project
프로젝트 이름을 브랜드명으로 설정합니다. 예: "CHAEUM Brand OS", "홍길동 스튜디오 AI"
02
Project Instructions 작성
이 공간이 핵심입니다. Claude가 이 프로젝트에서 어떻게 행동할지 지시합니다. 아래 템플릿을 참고하세요.
Project Instructions 기본 템플릿
당신은 [브랜드명]의 브랜드 AI 어시스턴트입니다.

## 역할
- 브랜드 가이드라인에 맞는 콘텐츠 작성
- 브랜드 톤앤매너 유지
- 실행 가능한 결과물 제공

## 반드시 지켜야 할 것
- 업로드된 BRAND.md의 말투와 금지 표현을 항상 준수할 것
- 모든 결과물은 브랜드 철학에 근거할 것
- 불확실한 내용은 추측하지 말고 확인을 요청할 것

## 결과물 형식
- 콘텐츠 요청 시 2-3개 옵션 제공
- 각 옵션에 간단한 이유 설명 포함
03
브랜드 파일 업로드
Project Knowledge → Add Content → 파일 업로드. BRAND.md, 제품 설명서, 타겟 페르소나 문서를 올려두면 Claude가 항상 참고합니다.
팁 — 파일은 최대 5개까지 올릴 수 있습니다. 핵심 문서만 간결하게 정리해서 올리는 게 효과적입니다.
2.4
Claude 모델 선택

Claude는 여러 버전이 있습니다. 브랜드 운영 목적에 따라 선택이 달라집니다.

Claude Sonnet — 일상 브랜드 운영용
콘텐츠 작성, 캡션 생성, 이메일 초안 등 반복적인 업무에 적합합니다. 속도와 품질의 균형이 가장 좋습니다. 채움이 가장 자주 쓰는 모델입니다.
Claude Opus — 전략적 사고가 필요할 때
브랜드 전략 수립, 복잡한 분석, 긴 문서 요약 등 깊은 사고가 필요한 작업에 사용합니다. 속도는 느리지만 결과물의 깊이가 다릅니다.
채움의 추천 방식
일상적인 콘텐츠 작업은 Sonnet, 분기별 브랜드 전략 수립이나 중요한 카피라이팅은 Opus를 씁니다. 두 가지를 상황에 따라 전환하는 습관을 들이세요.
2.5
세팅이 잘 됐는지 테스트하는 법

Projects 세팅을 마쳤다면 3가지 테스트로 확인합니다.

테스트 1 — 브랜드 인식 확인
"우리 브랜드를 한 줄로 소개해줘"라고 입력합니다. 업로드한 BRAND.md의 내용을 반영한 답변이 나오면 성공입니다. 엉뚱한 답변이 나오면 Project Instructions를 더 구체적으로 수정하세요.
테스트 2 — 말투 확인
"신제품 인스타그램 캡션 써줘"라고 입력합니다. 브랜드 말투와 톤이 맞는지 확인합니다. 맞지 않으면 BRAND.md의 예시 문장을 더 추가하세요.
테스트 3 — 금지 표현 확인
결과물에 BRAND.md에서 금지한 표현이 포함됐는지 확인합니다. 포함됐다면 금지 표현 섹션을 더 명확하게 작성하고 예시를 추가합니다.
결론

Projects 세팅은 한 번만 제대로 하면 됩니다. 이후 모든 작업에서 브랜드 일관성이 자동으로 유지됩니다.

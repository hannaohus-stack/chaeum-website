Foundation Series — F-03

MCP 연결 방법

읽는 시간 약 15분 · 무료 공개 · 다음 세션 → F-04 AI Tools 전체 지도

Claude는 기본적으로 대화 창 안에서만 작동합니다. 하지만 MCP(Model Context Protocol)를 연결하면 Notion, Google Drive, Gmail, 캘린더 같은 외부 도구와 직접 연동됩니다. 브랜드 DB를 Claude가 실시간으로 읽고, 쓰고, 관리할 수 있게 되는 것입니다.

3.1
MCP란 무엇인가

MCP는 Anthropic이 2024년 말 공개한 오픈 프로토콜입니다. 쉽게 말하면 "Claude가 외부 도구를 직접 사용할 수 있게 해주는 연결 규격"입니다.

MCP 이전과 이후
MCP 이전 — Claude에게 "우리 노션 브랜드 DB 업데이트해줘"라고 해도 직접 접근이 불가능했습니다. 내용을 복사해서 붙여넣는 수작업이 필요했습니다.

MCP 이후 — Claude가 직접 Notion에 접속해 페이지를 읽고, 수정하고, 새 내용을 추가할 수 있습니다. 사람 없이 자동으로 작동하는 브랜드 시스템의 핵심입니다.
3.2
브랜드 운영에 유용한 MCP 목록

수백 가지 MCP 서버가 있지만 브랜드 운영에 실제로 쓰이는 것은 다음과 같습니다.

문서 관리
Notion MCP
브랜드 DB, 콘텐츠 캘린더, 아이디어 노트를 Claude가 직접 읽고 씁니다.
채움 추천
파일 저장
Google Drive MCP
Drive에 저장된 브랜드 가이드라인, 사진, 문서를 Claude가 직접 접근합니다.
채움 추천
이메일
Gmail MCP
고객 문의 이메일을 읽고 브랜드 톤으로 초안을 작성합니다.
일정
Google Calendar MCP
콘텐츠 발행 일정을 자동으로 캘린더에 등록합니다.
검색
Brave Search MCP
실시간 시장 트렌드와 경쟁사 정보를 검색해 브랜드 전략에 활용합니다.
개발
GitHub MCP
웹사이트 코드를 직접 수정하고 배포합니다. 기술 브랜드에 유용합니다.
3.3
Claude Desktop에서 MCP 연결하기

MCP는 현재 Claude Desktop 앱에서 설정할 수 있습니다. 웹 브라우저 버전은 일부 MCP만 지원합니다.

주의 MCP 설정은 약간의 기술적 작업이 필요합니다. JSON 파일을 편집하는 과정이 포함됩니다. 어렵게 느껴지면 이 세션을 북마크해두고 나중에 돌아오세요. Foundation F-01, F-02만으로도 브랜드 운영 자동화의 80%는 가능합니다.
01
Claude Desktop 앱 설치
claude.ai/download에서 Mac 또는 Windows 버전을 다운받습니다. 설치 후 기존 계정으로 로그인합니다.
02
설정 파일 열기
Mac: 터미널에서 아래 명령어 입력
Windows: %APPDATA%\Claude\claude_desktop_config.json 파일 열기
Mac — 설정 파일 열기
open ~/Library/Application\ Support/Claude/claude_desktop_config.json
03
MCP 서버 정보 추가
파일을 열면 JSON 형식의 설정이 보입니다. mcpServers 항목에 사용할 MCP 정보를 추가합니다.
Notion MCP 연결 예시
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "NOTION_API_KEY": "여기에_노션_API_키_입력"
      }
    }
  }
}
04
Claude Desktop 재시작
설정 파일 저장 후 Claude Desktop을 완전히 종료하고 다시 실행합니다. 좌측 하단에 연결된 MCP 도구 아이콘이 표시되면 성공입니다.
팁 — 연결이 안 될 경우 Node.js가 설치되어 있는지 확인하세요. node.js 공식 사이트에서 LTS 버전을 설치하면 됩니다.
3.4
Notion API 키 발급받기

MCP 연결에는 각 서비스의 API 키가 필요합니다. 가장 많이 쓰는 Notion 기준으로 설명합니다.

01
notion.so/my-integrations 접속
Notion 계정으로 로그인 후 "New integration" 클릭합니다.
02
Integration 이름 설정
"Claude Brand OS" 등 알아보기 쉬운 이름으로 설정합니다. Submit 클릭 후 Internal Integration Token 복사합니다.
03
Notion 페이지에 권한 부여
Claude가 접근할 Notion 페이지 우측 상단 ••• → Connections → 방금 만든 integration 추가합니다. 이 페이지의 하위 페이지도 자동으로 접근 가능합니다.
팁 — 브랜드 DB 전체를 담은 상위 페이지 하나에 권한을 주면 하위 모든 페이지에 Claude가 접근할 수 있습니다.
3.5
연결 후 실제로 쓰는 법

MCP 연결이 완료되면 Claude에게 자연어로 지시하면 됩니다. 복잡한 명령어가 필요 없습니다.

Notion 연결 후 실제 사용 예시
"우리 브랜드 DB에서 이번 달 콘텐츠 캘린더 읽어줘"
"방금 작성한 인스타 캡션 3개를 노션 콘텐츠 캘린더에 추가해줘"
"브랜드 가이드라인 페이지에서 금지 표현 목록 찾아줘"
채움이 MCP를 쓰는 방식
매주 월요일 아침, Claude에게 "이번 주 콘텐츠 캘린더 노션에서 읽고 각 채널별 초안 작성해줘"라고 하면 — 노션을 읽고, 초안을 작성하고, 다시 노션에 저장합니다. 사람은 검토만 합니다.
결론

MCP는 Claude를 단순 채팅 도구에서 실제 브랜드 운영 시스템으로 바꾸는 핵심입니다. 처음엔 어렵게 느껴져도 한 번 연결해두면 이후 운영이 완전히 달라집니다.
다음 세션에서는 Claude 외 브랜드 운영에 필요한 AI 툴 전체를 한눈에 정리합니다.

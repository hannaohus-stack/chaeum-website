System Series — S-01

AI에게 내 브랜드
말투 이식하기

읽는 시간 약 15분 · 무료 공개 · 결과물 — BRAND.md 템플릿

AI에게 "우리 브랜드 톤으로 써줘"라고 말해본 적 있나요? 결과가 뻔하고 어딘가 어색했을 거예요. 이건 AI가 나쁜 게 아닙니다. 당신이 브랜드를 제대로 설명하지 않았기 때문입니다.

이 세션에서는 Claude가 내 브랜드의 언어를 완전히 이해하도록 만드는 법을 알려드립니다. 한 번 설정하면 이후 모든 결과물에 자동으로 적용됩니다.

1.1
왜 AI는 내 브랜드 말투를 모르는가

AI는 전 세계 텍스트를 학습했습니다. 그래서 "일반적인" 글을 잘 씁니다. 하지만 당신의 브랜드는 일반적이지 않습니다.

설명 없이 요청했을 때
"안녕하세요! 오늘은 특별한 상품을 소개해드리려고 해요. 많은 관심 부탁드립니다 😊"
브랜드를 학습시킨 후
"오늘 입고된 단 12병. 제주 서귀포 단일 농장, 콜드프레스 착즙. 매주 화요일만 열립니다."

차이가 보이나요. 같은 AI가 같은 제품을 소개하는데 결과가 완전히 다릅니다. 차이는 오직 하나 — 브랜드 정보를 어떻게 전달했는가입니다.

1.2
BRAND.md란 무엇인가

BRAND.md는 브랜드의 언어 설계서입니다. Claude가 읽고 이해할 수 있는 형식으로 작성된 문서로, 한 번 Projects에 올려두면 이후 모든 대화에 자동 적용됩니다.

BRAND.md에 담기는 것들
브랜드 철학과 핵심 가치 / 타겟 고객의 특성과 언어 / 사용하는 단어와 사용하지 않는 단어 / 문장 길이와 호흡 / 이모지 사용 여부 / 금지 표현 목록
왜 마크다운(.md) 형식인가
Claude는 구조화된 텍스트를 더 정확하게 이해합니다. 마크다운의 헤딩과 리스트 구조가 "이건 규칙이야"라는 신호를 명확하게 전달합니다. Word나 PDF보다 훨씬 효과적입니다.

1.4
Claude Projects에 이식하는 법

BRAND.md를 작성했다면, 이제 Claude에게 학습시킬 차례입니다.

Step 1 — Claude Projects 열기
claude.ai 접속 → 좌측 사이드바 "Projects" 클릭 → "New Project" 생성 → 프로젝트 이름을 브랜드명으로 설정
Step 2 — BRAND.md 업로드
Project Settings → "Add content" → BRAND.md 파일 업로드. 파일이 아니라면 내용을 직접 붙여넣기해도 됩니다.
Step 3 — 시스템 프롬프트 설정
Project Instructions에 아래 문장을 추가하세요.

"위 BRAND.md를 완전히 숙지하고, 모든 결과물을 브랜드 가이드라인에 맞게 작성하세요. 브랜드 말투와 금지 표현을 반드시 지키세요."
1.5
잘 됐는지 테스트하는 법

이식이 완료됐다면 아래 3가지 테스트로 확인해보세요.

테스트 1 — 인스타그램 캡션 요청
"신제품 출시 인스타그램 캡션 3개 써줘" — 결과물이 브랜드 말투와 맞는지 확인. 어색하다면 BRAND.md의 예시 문장을 더 추가하세요.
테스트 2 — 금지 표현 확인
결과물에 금지 표현이 포함됐는지 확인. 포함됐다면 BRAND.md의 금지 표현 섹션을 더 구체적으로 작성하세요.
테스트 3 — 브랜드 설명 요청
"우리 브랜드를 한 줄로 소개해줘" — Claude가 브랜드 철학을 제대로 이해했는지 확인할 수 있는 가장 빠른 방법입니다.
